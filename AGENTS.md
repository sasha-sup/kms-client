# kms-client

## Purpose

KMS (Key Management Service) server and client for Talos Linux LUKS disk encryption.
Provides gRPC-based Seal/Unseal operations for encryption keys, a dead-man's switch
heartbeat system with lease-based node blocking, and an HTTP heartbeat agent for
Kubernetes DaemonSet deployment. Originally from Siderolabs, adapted for internal use.

### Architecture Overview

```
┌──────────────┐  gRPC (Seal/Unseal)             ┌──────────────┐
│  Talos Node  │ ──────────────────────────────► │  KMS Server  │
│              │                                 │  :4050 gRPC  │
└──────────────┘                                 │  :4051 HTTP  │
                                                 │  :2112 metrics│
┌──────────────┐  HTTP POST /heartbeat           │              │
│  kms-agent   │ ──────────────────────────────► │              │
│  (DaemonSet) │  HMAC-SHA256 signed             └──────────────┘
└──────────────┘
```

- **kms-server**: gRPC server providing AES-GCM Seal/Unseal for LUKS keys, lease-based
  heartbeat validation, dead-man's switch background monitor, and HTTP admin/heartbeat API
- **kms-agent**: HTTP heartbeat agent (DaemonSet, HMAC-signed POST /heartbeat)
- **LeaseStore**: File-backed JSON persistence for node lease records (no external DB)

### Key Integration Points

- `pkg/server/server.go` — gRPC handlers (Seal, Unseal, Heartbeat)
- `pkg/server/http.go` — HTTP handlers (POST /heartbeat, admin endpoints)
- `pkg/server/lease_store.go` — Node state persistence and lease management
- `pkg/server/deadman.go` — Background goroutine blocking timed-out nodes
- `cmd/kms-server/main.go` — Server wiring and startup
